#1 Wie diese Website funktioniert

Absolute peak webdev talking!

Ich bin kein profi! Das hier ist alles aus meinem eigenen Wissen.. manche Dinge sind vielleicht komisch oder falsch beschrieben, nagel mich darauf bitti nicht fest :(

Ausserdem ich bin unfähig im schreiben und wahrscheinlich gibt es wieder Arsch viele Rechtschreibfehler.. naja.

Ich will hier gar nicht auf die Technischen Details eingehen, wie Websites und Server und all das funktionieren, sondern nur ganz sweet kurz erklären, wie ich das hier gemacht habe :D

Was ist eine Website??

Also ganz grundsetzlich: Eine Website ist eine "Webpage", die mit einem Domain-Namen auf irgendeinem web Server gehosted wird.

Diese Website besteht aus verschiedenen Webpages und liegt mit dem Domain-Namen "throwawayflookie.github" auf einem Server von GitHub. (Warum der Name so kacke ist und warum GitHub erkläre ich dir gleich)

Wie genau funktioniert diese Website technisch?

Wie bereits gesagt, wird diese Website auf GitHub gehosted.
GitHub ist primär eine Website für Entwickler und Programmierer auf der Code gespeichert, editiert, ausgetauscht und zusammen bearbeitet werden kann.

GitHub bietet aber auch den Service "GitHub pages" an, wo Projekte als Websites verwendet werden können.

Dieses project ist also auf GitHub und deren Server haben es gespeichert. Den Code kannst du dir sogar auf GitHub komplett anschauen, wenn du möchtest! GitHub

Also warum mache ich das so?

Server kosten Geld.. und wie du weisst, habe ich weniger als kein Geld.
Man muss monatlich Geld zahlen, um Server zu mieten und man muss eine Domain kaufen.
Dadurch, dass ich diesen kostenlosen Service benutze habe ich aber leider keine custom Domain (deswegen da oben diese hässliche Adresse) und ich habe keine volle Kontrolle über die Server, was zu einem weiteren Problem führt...

Static vs. Dynamic Websites

Das hier ist eine static Website, was heisst, dass es keine Serverlogik gibt.

Static Websites benutzen nur HTML und CSS, werden also nicht programmiert.
Weil du selber auch weisst wie man Websites schreibt, erkläre ich dir jetzt nicht, was diese Sprachen sind, aber es ist eben nur Text und Bilder.
Technically benutze ich auch JavaScript, aber GitHub lässt das halt einfach kostenlos zu, keine Ahnung..

Fun fact, ich habe bei meiner Recherche auf Wikipedia eine Software namens "Hugo" gefunden. Keine Ahnung was genau das ist, aber fand ich lustig.

Warum ist das jetzt ein Problem?

Es gibt Features, die ich einfach technisch nicht machen kann.

Also wie ich schon meinte bleibt die Domain leider so hässlich, aber das alleine ist jetzt nicht wirklich schlimm..
Was aber wirklich ein problem ist, ich kann keine Serverlogik programmieren.

Dass es keine serverseitige Logik gibt, heisst dass es nichts gibt, was im Hintergrund Daten verarbeitet oder dynamisch Inhalte erzeugt.
Stattdessen besteht die Seite aus festen Dateien, die ich geschrieben habe. Wenn du eine Seite öffnest, wird sie einfach so geladen, wie ich sie vorher erstellt habe.

In einem Online Shop kannst du zum Beispiel Produkte in deinen Warenkorb legen, um sie später zu bestellen. Dieses abspeichern wird dann vom Server übernommen, der vorher dafür programmiert wurde.

Was für Features kann ich also nicht machen?

Ich hatte ja bereits von einer Suggestion Box gesprochen. Meine Idee wäre, dass du ein Formular bekommst auf der Website, wo du irgendeine Idee reinschreiben kannst und wenn du fertig bist, wird es auf einem Server gespeichert, den ich dann selbst auslesen kann.

Naja, was braucht man dafür? Serverlogik
Vielleicht finde ich dafür noch eine andere Lösung oder ich kaufe mir eine Domain und miete Server!

Wie sieht meine höchst komplexe Arbeit an dieser Website aus??

Ich meinte ja bereits, dass ich die Datein, die ich für dich schreibe bei GitHub speichere, aber das Bearbeiten passiert auf meinem PC.

Wenn ich jetzt also eine Idee für ein neues Feature schreiben möchte, öffne ich Visual Studio Code (meinen Texteditor), mache eine neue HTML Datei und schreibe den Code.

Wenn ich dann mit meinem Code zufrieden bin, muss ich nur ein paar commands in meinem Terminal eingeben und es auf GitHub bringen.

git add ./
git commit -m "Super cooles neues Feature :D"
git push origin master

Ohne jetzt zu technisch zu werden (vielleicht mache ich das mal in einem eigenen info dump), der Code den ich auf meinem PC geschrieben habe, wird jetzt einfach auf GitHub hinzugefügt.

GitHub sieht es gab einen neuen push und die Website wird geupdated! Zack neues Feature, dass du dir auf deinem Handy anschauen kannst :D
